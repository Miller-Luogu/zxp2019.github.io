什么是 Gitalk

Gitalk 是基于 Github Issues 的一款评论插件。被广泛应用于 Github Pages 搭建的博客中。

Gitalk 支持 Markdown 语法，并且整个 UI 都特别美观。

Gitalk 不需要留邮箱地址，也不需要注册账号。他是基于 Github 账号的，可以直接通过 Github 账号发东西。

Gitalk 的本质实际上是 Issue，在一个页面安装 Gitalk 后，会发现博客多出来一个 Issue，而 Gitalk 上显示的评论实际上就是 Issue 中的 comment。所以不难理解为什么可以用 Markdown 了。

Gitalk 目前在 Github 上开源。Repo 地址：https://github.com/gitalk/gitalk/

申请 Github Application

使用 Gitalk 需要先申请 Github Application。

点这里，会进入申请 Github App 界面。

  • Application Name 那里随便填，比如填 My Blog 就可以。
  • Homepage URL 那里填你博客主站的地址（比如我就可以填 https://zxp2019.github.io ）
  • Application Description 可以不填。
  • Authorization Callback URL 也填你博客主站的地址（和 Homepage URL 相同）

然后看见那个 AC 色的按钮了吗？点它！

之后你进入了一个网页（截的图找不到了 qwq）。里面有两个重要信息：Client ID 和 Client Secret（都是字符串）。这两个信息一定要保存下，写这篇文章时我再回去就找不到这两个信息了。

这样你就申请好 Github Application 了。

在博客代码中使用 Gitalk

将下面的代码复制一下。

<!-- 导入Gitalk 支持文件-->
<link rel="stylesheet" href="https://unpkg.com/gitalk/dist/gitalk.css">
<script src="https://unpkg.com/gitalk@latest/dist/gitalk.min.js"></script> 
<!---->
<div id="gitalk-container"></div>    <script type="text/javascript">
    var gitalk = new Gitalk({

    // gitalk的主要参数
        clientID: '你的 Client ID',
        clientSecret: '你的 Client Secret',
        repo: '你博客的 repo 名',
        owner: '你的 Github 用户名',
        admin: ['你的 Github 用户名'],
        id:window.location.pathname,

    });
    gitalk.render('gitalk-container');
</script>

需要注意几点：

  • 前面的 <link> 和 <script> 可以放在 head 里。
  • 后面的几项参数一定是包含单引号的，不能去掉。
  • 那个 <div> 需要放到你要插入评论区的位置。

初始化评论区

在评论区设置之初，Gitalk 会自动生成一个 Issue。所以第一次加载会比较慢。

一定要找个网好的时段！！！

然后刷新你放置评论区的那个页面，会看到类似下面的状况。

然后发东西就可以了！

如果遇到 Network Error 或者下面的状况，如果确保自己配置的没问题的话，多刷新几次。如果你的博客绑定了其他域名，可以看看这篇文章。

发完第一条评论后去 Issue 区，可以看见新创建了一个 Issue，其名字就是你放评论区那个页面的 title。

而这个 Issue 里的内容其实就是你评论的内容。

所以当网不好加载不出来 Gitalk 时，不妨直接找到这个 Issue 进行 Comment。
