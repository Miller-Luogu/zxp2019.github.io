Part I. What Is Github

官网：

Git is a free and open source distributed version control system.

百度百科：

Git 是一个免费的开源分布式版本控制系统。

简单的来说，Git 就是 Github 的内核。而 Github 就是基于 Git 的一个帮助开发者开源并控制版本的系统。Github 在 Git 的基础上增加了图形化界面，便于操作。

Part II. Why we use Github?

Github 良好的功能非常方便开发 ，因此很多开发者都在使用。当然，每个账号还能使用 Github Pages 免费搭建一个 github.io 的网站！这比我们买域名和服务器要方便多了。

因此，大部分 OIer 去 Github 可能主要是为了搭自己的博客（当然也有的是去开发的）。所以这里以建立博客为例说明（其他的类似）。

Part IV. 创建 repo

Github 中一个最基本的元素就是 repository(repo) 了。它可以理解为“项目”。

我们点击 “Create a repository”。随后出现这个经典的页面：

我们需要创建的是博客。所以请在 “Repository name” 那里填入 “xxx.github.io”，xxx为你的用户名（尽量要这么填，好像听说 xxx 那里不填自己的用户名会不行）。如果你要创建的是个普通项目，那么这里请填写项目名字。

“Owner” 那里如果是你本人就不用管。

“Description” 是你对这个项目的描述。会以一行小字显示在 repo 的下面。要用简练的语言描述。在这里可以填写 “My github.io blog.”

“Public” 表示你这个项目是公开的。一定要选这个选项。“Private” 是非公开项目，要花钱而且达不到目的。

“Initialize this repository with a README” 这个选项表示是否同时创建一个 README 文档。这个文档是用来介绍你的项目的，采用 Markdown 格式（注意，README 通常是详细的介绍，与 Description 不同）。创建博客不需要 README。但是其他的项目建议创建 README，以便让别人理解你这个项目的意义。

底下的两个选项就保持默认设置即可。

Part V. repo 操作

这个 repo 就像你电脑一样，也是有文件系统的。初始时显示的就是根目录。

上传文件

这时我们点击右上方的 “Upload Files”，进入如下界面：

如果你上传单个文件，那么直接点 “Choose your files” 就 ok，但是如果你要上传文件夹啥的，必须把文件夹 拖拽 到那个框框里。

然后这里就是上传的文件的列表了。

可以在 Commit Changes 里写一下自己本次操作的目的，如果不写系统会自动生成。

然后点下面的绿色按钮提交。提交之后需要等一会才能看到结果。

修改文件

更多的时候我们是更新原有文件。进入到相应的目录下，点击要更新的文件，会出现文件的显示界面：

点击 History 右边的第二个铅笔图标可以跳转到更改页面，更改后记得 commit changes。点击垃圾桶图标可以删除，也要记得 commit changes。

修改后会跳转到新的文件显示界面，此时会到 repo 的主页，会发现在上面的蓝条中（如下图）出现了你上一次更改的描述。

点击描述会进入更改的详细信息（有本次更改和上次的对比）。

点击描述即可看到本次更改的详细信息。

branch

branch（分支）是用来管理这个项目的不同版本的。默认分支是 master 分支。在 commit changes 的时候会有一个存储到分支选项。

复制 / 下载到本地

你可以把别人的 repo clone 到你的本地下，方法是点击 repo 主页右上角绿色的按钮“clone or download”。

你还可以直接下载别人的 repo 到本地，方法是 “clone or download”->“download ZIP”。然后会以压缩包形式下载。如果本地没有配置 Git 需要使用 download。

fork

可以通过 fork 把别人的 repo 拷贝一个副本到你的账号下。点击最右上角 “Unwatch”，“Star”，“Fork” 中的 “Fork”。

Pull Requests

如果你想为别人的 repo 做贡献，需要先将他的 repo fork 到自己这，然后增加你的贡献内容后，利用 Pull Request 把你的修改后的 repo “给回去”。

收藏

Github 也有“收藏”功能！如果觉得一个 repo 好可以 点击“Star”来给收藏它！

Issue

发现了 repo 的问题？有建议？可以通过发 Issue 的方式告知 repo 的管理员。另外，你还可以评论 Issue。

Wiki

想帮助别人更好的了解你的 repo？使用 Wiki 功能。点击上面那一栏中间的 Wiki。

Settings

点击 repo 主页上面的那一栏中最右边的 Settings 可以设置你的 repo。下面给出一些常用设置：

  • 改名：Options->Repository name
  • 是否附带 wiki：Options->Features->Wikis
  • 是否附带 issue：Options->Features->Issues
  • 选择博客主题（如果你不会 HTML 可以来这里用人家预置的主题）：Options->Github Pages->Theme Chooser
  • 变为私有 repo：Options->Danger Zone->Make this repository private
  • 删除 repo：Options->Danger Zone->Delete this repository
  • 添加分支：Branch->Default Branches

Part VII 关于用 Github Pages 搭建自己的博客

应某 OIer 要求，加一段，专门介绍这个。

首先，自己的博客应该是名称为 xxx.github.io的一种 repo，是一种特殊的 repo。

这种 repo 一般来说不需要添加 LICENSE 和 README.md 。

然后建好 repo 之后就把它当做是你的服务器，该怎么添加文件就怎么添加文件即可。HTML，CSS，JS 均可正常用。

然后别人输入 xxx.github.io 就能访问到你的网页。

但是需要注意几点：

  • Upload 文件之后需要按 Ctrl + F5 刷新那个页面，不然就要等好久才能显示出来。

用 Github Pages 的好处：

  • 不需要买域名，也不需要备案，还不需要自己配置服务器。
  • 不需要专门提防攻击，Github 服务器已经有防止攻击的系统了。
  • 使用起来很方便。

但是 Github 的唯一（永恒不变）的缺点就是...

太慢了！！！
